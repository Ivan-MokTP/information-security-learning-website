Secure Socket Layer
Security over Internet Communication

Apart from identifying internet users, another kind of authentication called Secure Socket Layer(SSL) are also widely adopted nowadays. SSL provides a secure channel between two machines or devices operating over the internet or an internal network. As such, only the sender and receiver in the channel can have access to the protected information contained within.

SSL consists of 2 important components - certificate and keys.

  • - Certificate: Contains basic information and a digital signature that can identify the client or server.
  • - Keys: Used to establish a secured connection
Mutual Authentication

SSL is usually a Mutual Authentication (or Two-Way Authentication) process. It means both server and client will have to send its certificate to another entity.

  1. Client connects to the web server secured with SSL. Then client requests the server to identfy itself.
  2. Server sends a copy of its certificate and its public key. Client responds by sending a copy of its certificate to the server. (Here completes the Mutual Authentication process.)
  3. Client validates the server's certifcate. Then client creates a temporary session key using server's public key, and sends it to server.
  4. Server decrypts the session key using its own private key. Now both entities conrtain the same session key. They now use the common session key to encrypt data and transmit through the channel.
