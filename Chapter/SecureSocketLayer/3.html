Secure Socket Layer
Mutual Authentication

SSL is usually a Mutual Authentication (or Two-Way Authentication) process. It means both server and client will have to send its certificate to another entity. Only when both entities validate the certificate of the other entity will the secured channel be established.

In simple word:
Server Authentication + Client Authentication = Mutual Authentication

  1. Client connects to the web server secured with SSL. Then client requests the server to identfy itself.
  2. Server sends a copy of its certificate and its public key. Client responds by sending a copy of its certificate to the server. (Here completes the Mutual Authentication process.)
  3. Both entities validate the certifcate of another entity. Then client creates a temporary session key using server's public key, and sends it to server.
  4. Server decrypts the session key using its own private key. Now both entities contain the same session key. They now use the common session key to encrypt data and transmit through the channel.
