Secure Socket Layer
Setting Up SSL

Before using SSL, both sides have to set up an initial communication in order to generate a seesion key. The seesion is shared by and only known to the two sides. After that the both sides will encrypt transmitting data with session key, hance securing all the data within.

In normal SSL, only Server Authentication is needed. The diagram is as follow:

  1. Client connects to the web server secured with SSL. Then client requests the server to identfy itself.
  2. Server sends a copy of its certificate and its public key.
  3. Client validates the server's certifcate. Then client creates a temporary session key using server's public key, and sends it to server.
  4. Server decrypts the session key using its own private key. Now both entities contain the same session key. They now use the common session key to encrypt data and transmit through the channel.
